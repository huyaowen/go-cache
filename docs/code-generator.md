# 代码生成器使用指南

## 问题背景

在 Go 语言中，由于不支持运行时动态实现接口，使用缓存代理时需要手动实现接口包装器：

```go
// 手动实现所有方法 ❌
type decoratedUserService struct {
    decorated *proxy.DecoratedService[*userService]
}

func (d *decoratedUserService) GetUser(id int64) (*model.User, error) {
    results, err := d.decorated.Invoke("GetUser", id)
    // ... 重复的样板代码
}
```

**问题**：
- 每个服务都需要手动实现包装器
- 代码重复，维护成本高
- 接口变更时需要手动更新

## 解决方案：代码生成器

使用 `go-cache-gen` 代码生成器自动生成接口包装器。

### 使用方法

#### 1. 在服务接口文件中添加生成指令

```go
//go:generate go run ../../../cmd/generator/main.go .

package service

// UserServiceInterface 用户服务接口
type UserServiceInterface interface {
    GetUser(id int64) (*model.User, error)
    CreateUser(user *model.User) (*model.User, error)
    // ...
}
```

#### 2. 运行代码生成器

```bash
# 方式 1: 使用 go generate
cd examples/gin-web/service
go generate

# 方式 2: 直接运行
go run ../../../cmd/generator/main.go .
```

#### 3. 生成的代码

生成器会创建 `generated_wrapper.go`：

```go
// Code generated by go-cache-gen. DO NOT EDIT.

package service

import (
    "fmt"
    "github.com/coderiser/go-cache/pkg/proxy"
    "github.com/coderiser/go-cache/examples/gin-web/model"
)

// DecoratedUserService 自动生成的装饰服务包装器
type DecoratedUserService struct {
    decorated *proxy.DecoratedService[*userService]
}

// NewDecoratedUserService 创建装饰后的服务实例
func NewDecoratedUserService(decorated *proxy.DecoratedService[*userService]) *DecoratedUserService {
    return &DecoratedUserService{decorated: decorated}
}

// GetUser 获取用户（通过代理调用）
func (d *DecoratedUserService) GetUser(id int64) (*model.User, error) {
    results, err := d.decorated.Invoke("GetUser", id)
    if err != nil {
        return nil, err
    }
    if len(results) == 0 {
        return nil, fmt.Errorf("no result returned")
    }
    result, ok := results[0].(*model.User)
    if !ok {
        return nil, fmt.Errorf("unexpected result type: %T", results[0])
    }
    return result, nil
}

// ... 其他方法自动生成
```

#### 4. 在 init 中使用

```go
func InitCache() {
    // 创建原始服务
    rawService := NewUserService()
    
    // 使用泛型装饰
    decorated := proxy.SimpleDecorateWithManager(rawService, cacheManager)
    
    // 使用生成的包装器实现接口
    UserService = NewDecoratedUserService(decorated)
}
```

## 生成的文件

代码生成器会生成两个文件：

1. **`.cache-gen/auto_register.go`**: 注解注册代码
2. **`service/generated_wrapper.go`**: 接口包装器代码

## 优势

✅ **类型安全**: 编译时检查，避免运行时错误  
✅ **零运行时开销**: 直接方法调用，无需反射  
✅ **自动维护**: 接口变更时重新生成即可  
✅ **符合 Go 惯例**: 类似 `mockgen`, `stringer` 的标准做法  
✅ **接口抽象**: handler 继续使用接口类型，保持解耦  

## 注意事项

1. **生成的代码不要手动修改** - 每次运行会覆盖
2. **接口变更后重新生成** - `go generate ./...`
3. **提交生成的代码** - 确保其他开发者可以使用
4. **命名约定**: 接口名应为 `XxxInterface` 格式（如 `UserServiceInterface`）

## 替代方案对比

### 方案 A: 代码生成（推荐）✅

- 类型安全
- 零运行时开销
- 需要运行 `go generate`

### 方案 B: 直接使用 DecoratedService + Invoke

```go
type UserHandler struct {
    userService *proxy.DecoratedService[*userService]
}

func (h *UserHandler) GetUser(c *gin.Context) {
    id, _ := strconv.ParseInt(c.Param("id"), 10, 64)
    results, err := h.userService.Invoke("GetUser", id)
    user := results[0].(*model.User)  // 需要类型断言 ❌
    c.JSON(200, user)
}
```

- 无需包装器
- 需要类型断言（不安全）
- 失去接口抽象

### 方案 C: 手动实现包装器

- 类型安全
- 代码重复 ❌
- 维护成本高 ❌

## 总结

**推荐使用代码生成器**，它结合了类型安全、自动维护和 Go 语言最佳实践。
