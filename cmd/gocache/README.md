# gocache CLI

**代码生成工具** - 扫描 Go 代码中的缓存注解并自动生成包装器

---

## 📦 安装

```bash
go install github.com/coderiser/go-cache/cmd/gocache@latest
```

验证安装：

```bash
gocache --help
```

## 🚀 快速开始

### 1. 编写业务代码

```go
// service/user.go
package service

// UserServiceInterface 定义服务接口
type UserServiceInterface interface {
    GetUser(id int64) (*User, error)
    CreateUser(name, email string) (*User, error)
    DeleteUser(id int64) error
}

// userService 服务实现
type userService struct {
    db *sql.DB
}

func NewUserService(db *sql.DB) *userService {
    return &userService{db: db}
}

// @cacheable(cache="users", key="#id", ttl="30m")
func (s *userService) GetUser(id int64) (*User, error) {
    // 业务逻辑
}

// @cacheput(cache="users", key="#result.ID", ttl="30m")
func (s *userService) CreateUser(name, email string) (*User, error) {
    // 业务逻辑
}

// @cacheevict(cache="users", key="#id")
func (s *userService) DeleteUser(id int64) error {
    // 业务逻辑
}
```

### 2. 执行扫描

```bash
# 扫描当前目录
gocache scan

# 扫描所有包
gocache scan ./...

# 扫描指定目录
gocache scan ./service

# 详细输出
gocache scan -v ./service
```

### 3. 生成的代码

```go
// service/user_cached.go
// Code generated by gocache. DO NOT EDIT.

package service

import (
    "context"
    "time"
    
    "github.com/coderiser/go-cache/pkg/cache"
    "github.com/coderiser/go-cache/pkg/core"
    "github.com/coderiser/go-cache/pkg/spel"
)

// cachedUserService 缓存包装器
type cachedUserService struct {
    raw       *userService
    manager   core.CacheManager
    evaluator *spel.SpELEvaluator
}

// UserService 全局实例
var UserService UserServiceInterface

// NewCachedUserService 创建带缓存的服务实例
func NewCachedUserService(db *sql.DB) UserServiceInterface {
    raw := NewUserService(db)
    return &cachedUserService{
        raw:       raw,
        manager:   cache.GetGlobalManager(),
        evaluator: spel.NewSpELEvaluator(),
    }
}

// InitUserService 初始化全局实例
func InitUserService(db *sql.DB) {
    UserService = NewCachedUserService(db)
}

func (c *cachedUserService) GetUser(id int64) (*User, error) {
    ctx := context.Background()
    cache, _ := c.manager.GetCache("users")
    
    // 构建缓存 Key
    evalCtx := spel.NewEvaluationContext()
    evalCtx.SetArg("id", id)
    key, _ := c.evaluator.EvaluateToString("#id", evalCtx)
    
    // 查缓存
    if val, found, _ := cache.Get(ctx, key); found {
        return val.(*User), nil
    }
    
    // 调用原始方法
    result, err := c.raw.GetUser(id)
    if err != nil {
        return nil, err
    }
    
    // 写缓存
    cache.Set(ctx, key, result, 30*time.Minute)
    
    return result, nil
}

// ... 其他方法
```

### 4. 使用服务

```go
// main.go
package main

import (
    "your-project/service"
)

func main() {
    db := initDB()
    
    // 方式 1：初始化全局实例
    service.InitUserService(db)
    user, _ := service.UserService.GetUser(1)
    
    // 方式 2：使用构造函数
    userService := service.NewCachedUserService(db)
    user, _ = userService.GetUser(1)
}
```

## 📖 命令参考

### gocache scan

扫描 Go 文件中的缓存注解并生成包装器代码。

```bash
gocache scan [options] [directories]
```

**选项:**

| 选项 | 说明 | 默认值 |
|------|------|--------|
| `-force` | 强制全量扫描（忽略增量状态） | `false` |
| `-v` | 详细输出 | `false` |
| `-h` | 显示帮助 | - |

**示例:**

```bash
# 扫描所有包
gocache scan ./...

# 扫描指定目录
gocache scan ./service ./repo

# 强制全量扫描
gocache scan -force ./...

# 详细输出
gocache scan -v ./service
```

## 🎯 支持的注解

### @cacheable

缓存读取操作。

```go
// @cacheable(cache="缓存名", key="Key 表达式", ttl="过期时间")
// @cacheable(cache="users", key="#id", ttl="30m")
func GetUser(id int64) (*User, error)
```

**参数:**
- `cache`: 缓存名称（必需）
- `key`: 缓存 Key 的 SpEL 表达式（必需）
- `ttl`: 缓存过期时间（可选，默认 30m）
- `condition`: 缓存条件（可选）
- `unless`: 排除条件（可选）

### @cacheput

缓存更新操作。

```go
// @cacheput(cache="缓存名", key="Key 表达式", ttl="过期时间")
// @cacheput(cache="users", key="#result.ID", ttl="30m")
func CreateUser(name string) (*User, error)
```

**参数:**
- `cache`: 缓存名称（必需）
- `key`: 缓存 Key 的 SpEL 表达式（必需），支持 `#result` 变量
- `ttl`: 缓存过期时间（可选，默认 30m）

### @cacheevict

缓存清除操作。

```go
// @cacheevict(cache="缓存名", key="Key 表达式", before=true/false)
// @cacheevict(cache="users", key="#id")
func DeleteUser(id int64) error
```

**参数:**
- `cache`: 缓存名称（必需）
- `key`: 缓存 Key 的 SpEL 表达式（必需）
- `before`: 是否在方法执行前清除（可选，默认 false）

## 🎨 SpEL 表达式

### 支持的变量

| 变量 | 说明 | 示例 |
|------|------|------|
| `#id`, `#user` | 参数名 | `key="#id"` |
| `#p0`, `#p1` | 参数索引 | `key="#p0"` |
| `#0`, `#1` | 参数索引 (简写) | `key="#0"` |
| `#result` | 返回值 (仅 `@cacheput`) | `key="#result.ID"` |

### 表达式示例

```go
// 访问参数属性
// @cacheable(cache="users", key="#user.Id")

// 访问嵌套属性
// @cacheable(cache="orders", key="#order.Customer.Id")

// 字符串拼接
// @cacheable(cache="data", key="#prefix + ':' + #id")

// 条件表达式
// @cacheable(cache="data", key="#id", condition="#id > 0 && #id < 1000")
```

## ⚙️ 增量扫描

`gocache` 支持增量扫描，只处理修改过的文件，提高生成速度。

```bash
# 第一次扫描（全量）
gocache scan ./...

# 后续扫描（增量，只处理修改的文件）
gocache scan ./...

# 强制全量扫描
gocache scan -force ./...
```

**增量状态文件:** `.gocache/state.json`（自动生成）

## 🔍 常见问题

### Q: 生成的代码在哪里？

**A**: 在扫描目录中，与源文件同目录，文件名格式为 `xxx_cached.go`。

例如：`service/user.go` → `service/user_cached.go`

### Q: 注解不生效？

**排查步骤:**
1. 检查注解格式：`// @cacheable(cache="users", key="#id", ttl="30m")`
2. 检查是否执行了 `gocache scan ./...`
3. 检查生成的 `*_cached.go` 文件是否存在
4. 检查接口和实现是否匹配

### Q: 如何排除某些文件？

**A**: 以下文件会自动排除：
- `*_test.go` (测试文件)
- `*_cached.go` (生成的文件)
- `gocache_gen/` 目录
- `.cache-gen/` 目录

### Q: 如何自定义输出目录？

**A**: 当前版本输出到源文件同目录。如需自定义，请提交 issue。
