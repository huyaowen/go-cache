## 修复问题清单

根据测试报告，共修复了 3 个主要问题：

### 1. ✅ 注解类型支持不完整

**问题描述**: 
生成器仅支持 `@cacheable` 注解，不支持 `@cacheput` 和 `@cacheevict`，导致 40% 的注解组合测试失败。

**修复内容**:

#### 1.1 扩展 Annotation 结构体 (`extractor/extractor.go`)
```go
type Annotation struct {
    Type           string // Method name
    CacheName      string // Cache name
    Key            string // Cache key expression
    TTL            string // Time to live
    Condition      string // Condition expression (SpEL)
    Unless         string // Unless expression (SpEL)
    Before         bool   // Execute before method (for cacheevict)
    Sync           bool   // Synchronous execution
    AnnotationType string // Annotation type: cacheable, cacheput, cacheevict
}
```

#### 1.2 扩展 ExtractAnnotation 函数 (`extractor/extractor.go`)
```go
func ExtractAnnotation(comment string) *Annotation {
    // Match @cacheable, @cacheput, @cacheevict annotations
    re := regexp.MustCompile(`//\s*@(cacheable|cacheput|cacheevict)\s*\((.*)\)`)
    // ... 解析所有参数
}
```

**验证结果**: 
- ✅ 注解组合测试 5/5 通过（之前 2/5）
- ✅ 支持 condition 和 unless 参数
- ✅ 支持 before 和 sync 标志

### 2. ✅ Channel 类型方向信息丢失

**问题描述**: 
`<-chan` 被简化为 `chan`，只读 channel 语义丢失。

**修复内容**:

#### 2.1 更新 typeToString 函数 (`parser/parser.go`)
```go
case *ast.ChanType:
    // Preserve channel direction: <-chan, chan<-, or chan
    switch t.Dir {
    case ast.RECV:
        return "<-chan " + p.typeToString(t.Value)
    case ast.SEND:
        return "chan<- " + p.typeToString(t.Value)
    default:
        return "chan " + p.typeToString(t.Value)
    }
```

**验证结果**: 
- ✅ 复杂类型测试中 `<-chan *User` 正确生成
- ✅ Channel 方向信息完整保留

### 3. ✅ 缺少自动导入生成

**问题描述**: 
生成的代码未导入引用类型所在的包，导致生成的代码无法直接编译。

**修复内容**:

#### 3.1 扩展 ServiceInfo 结构体 (`extractor/extractor.go`)
```go
type ServiceInfo struct {
    Name       string
    ImplType   string
    Package    string
    ImportPath string
    Imports    map[string]string // Required imports (alias -> path)
    Methods    []*MethodInfo
}
```

#### 3.2 在 Parser 中收集导入信息 (`parser/parser.go`)
```go
func (p *Parser) ExtractServices(file *ast.File, annotations []*extractor.Annotation) []*extractor.ServiceInfo {
    // Collect imports from the file
    imports := make(map[string]string)
    for _, imp := range file.Imports {
        path := strings.Trim(imp.Path.Value, "\"")
        alias := ""
        if imp.Name != nil {
            alias = imp.Name.Name
        }
        if alias == "" || alias == "_" || alias == "." {
            parts := strings.Split(path, "/")
            alias = parts[len(parts)-1]
        }
        imports[alias] = path
    }
    // ...
}
```

#### 3.3 更新模板支持导入 (`generator/templates.go`)
```go
const wrapperTemplate = `// Code generated by generator; DO NOT EDIT.
package {{.Package}}

import (
    "generator/proxy"
{{- range $alias, $path := .Imports}}
    {{$alias}} "{{$path}}"
{{- end}}
)
// ...
`
```

#### 3.4 更新 Generator 合并导入 (`generator/generator.go`)
```go
func (g *Generator) GenerateWrapper(services []*extractor.ServiceInfo) ([]byte, error) {
    // Merge all imports
    allImports := make(map[string]string)
    for _, svc := range services {
        if svc.Imports != nil {
            for alias, path := range svc.Imports {
                if strings.HasPrefix(path, "generator/") {
                    continue
                }
                allImports[alias] = path
            }
        }
    }
    // ...
}
```

**验证结果**: 
- ✅ 生成的代码包含正确的 import 语句
- ✅ 支持多文件导入合并
- ✅ 自动过滤内部包导入
